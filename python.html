git教程

  • git init #初始化一个仓库
  • 添加文件至git仓库分为两步：
    • git add <file>将文件添加到暂存区
    • git add .添加全部文件
  • git commit -m 'say something'把暂存区的所有内容提交到当前分支
  • git status #随时掌握仓库当前的状态，可以知道某文件被修改，无法具体得知修改了什么
  • git diff <file> #可以查看某文件具体修改了什么内容
$ git diff readme.txt 
diff --git a/readme.txt b/readme.txt
index 46d49bf..9247db6 100644
--- a/readme.txt
+++ b/readme.txt
@@ -1,2 +1,2 @@
-Git is a version control system.
+Git is a distributed version control system.
 Git is free software.
  • git log #显示从最近到最远的提交日志，即git commit -m '提交日志'的内容
  • git log --pretty=oneline #简略显示提交内容

版本库回退

  • HEAD当前版本 HEAD^上一个版本 HEAD^^上上一个版本
  • git reset --hard HEAD^ #回退到上一个版本
  • git reflog #记录每一次命令，即命令历史
  • 欲知更多回退版本的技巧，参考版本回退
  • 电脑下的learngit文件夹为工作区（Working Directory），里面有一个.git的隐藏目录，为版本库(respository)，版本库下有一个暂存区(stage或者index)，还有Git为我们自动创建的第一个分支master，以及指向master的一个指针叫HEAD。

工作区回退

  • git checkout -- readme.txt#把readme.txt文件在工作区的修改全部撤销，也可用于把误删的文件恢复

暂存区回退

  • git reset HEAD <file> #可以把暂存区的修改撤销掉(unstage),
  • 然后再丢弃工作区的修改git checkout -- <file>

删除commit的文件

  • git rm test.txt
  • git commit -m 'remove test.txt'

添加远程仓库

  • git remote add origin git@github.com:hondajojo/learn.git
  • git push -u origin master #第一次推送master分支内容至远程仓库
  • git push origin master #此后，每次提交后，用此命令推送最新更改

tips

你也许还注意到，GitHub给出的地址不止一个，还可以用https://github.com/michaelliao/gitskills.git这样的地址。实际上，Git支持多种协议，默认的git://使用ssh，但也可以使用https等其他协议。 使用https除了速度慢以外，还有个最大的麻烦是每次推送都必须输入口令，但是在某些只开放http端口的公司内部就无法使用ssh协议而只能用https。

创建和合并分支

  • git branch #查看分支
  • git branch <name> #创建分支
  • git checkout <name> #切换分支
  • git checkout -b <name> #创建及切换分支，-b(branch)
  • git merge <name> #合并某分支到当前分支
  • git branch -d <name> #删除分支

  • 在本地创建ssh key，ssh-keygen -t rsa -C 'cm1130335361@gmail.com' #生成在/home/mako/.ssh文件下
